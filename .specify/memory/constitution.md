# FareFinder Constitution

## Principles

1. User-facing comparison flows must remain simple, fast, and transparent.
2. Fare and product results must clearly distinguish estimates from live prices.
3. Secrets and local configuration must never be committed.
4. Every feature change should include documentation and focused validation.
5. AGPLv3 licensing must be preserved across distributions.

## Quality Gates

Changes should pass linting, formatting, type checking, tests, coverage,
dependency audit, and secret scanning before merge.
